# 聚合规则

## 核心原则

**聚合前必须检查数据粒度**：当表粒度小于目标聚合粒度时，必须先去重再聚合。

## 聚合类型分类

### 类型一：直接聚合

**特征**：表的粒度与目标聚合粒度一致，或字段本身就是原子值。

**处理方式**：直接 SUM/COUNT/AVG

```sql
-- 示例：折扣金额直接 SUM
SELECT SUM(discount_amt) FROM table
```

### 类型二：去重聚合

**特征**：表粒度小于目标聚合粒度（如订单行级表统计订单数），字段在多条记录中重复。

**处理方式**：COUNT(DISTINCT field) 或 SUM(DISTINCT field)

```sql
-- 示例：订单数统计
SELECT COUNT(DISTINCT order_id) FROM order_line_table
```

### 类型三：粒度提升聚合（重点）

**特征**：
- 表粒度为"订单行级"或"明细级"
- 字段值在粒度更细的记录中重复（如整单金额在商品行中重复）
- 目标是统计更高粒度的汇总值

**处理方式**：必须先按更高粒度取 MAX/MIN，再做 SUM

**典型场景**：
| 场景 | 表粒度 | 字段含义 | 错误做法 | 正确做法 |
|------|--------|----------|----------|----------|
| 订单金额统计 | 订单行级 | 整单金额 | SUM(pay_amount) | SUM(MAX(pay_amount)) over order_id |
| 核销金额统计 | 券+订单+品类 | 整单金额 | SUM(pay_amount) | 先按 order_id 取 MAX 再 SUM |

**通用模板**：
```sql
-- 错误：直接聚合
SELECT dimension, SUM(amount_field)
FROM detail_table
GROUP BY dimension

-- 正确：先提升粒度再聚合
SELECT dimension, SUM(amount_field)
FROM (
    SELECT dimension, order_id, MAX(amount_field) AS amount_field
    FROM detail_table
    GROUP BY dimension, order_id
) t
GROUP BY dimension
```

## 识别规则

在生成聚合 SQL 时，按以下顺序检查：

1. **检查表粒度**：查看表说明中的"数据粒度"字段
2. **检查字段含义**：查看字段说明中的字段定义
3. **判断是否重复**：如果字段值在表粒度下会重复，使用类型三
4. **遵循样例 SQL**：指标说明中的样例 SQL 是最高准则

## 字段分类参考

| 字段类型 | 典型聚合方式 |
|----------|--------------|
| 数量类（qty, count） | 直接 SUM 或 COUNT(DISTINCT) |
| 金额类（amt, amount, sales） | 检查粒度，可能需要类型三 |
| 比例类（rate, ratio） | 通常需要先聚合分子分母再计算 |
| 标识类（id, no） | COUNT(DISTINCT) |

如果不确定，优先查看指标说明中的样例 SQL。
